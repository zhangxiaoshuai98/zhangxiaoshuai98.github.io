本篇是springboot2.x快速上手的第二篇记录，继续学习，继续前进。

Servlet还是那个Servlet——整合Servlet

springboot去掉了所有xml，程序的入口变成了main，这和我们的普通java项目一样(实际上就是一个jar项目)，那我们首先考虑到，Servlet怎么办，Filter怎么办，Listener怎么办？？？

别担心，这些都和外部Tomcat启动时一样，Servlet还是那个Servlet。

之前我们使用@WebServlet、@WebFilter、@WebListener，现在依然是这样，只不过之前这些解析是由Tomcat自动解析，而现在交给了springboot。因此，我们需要在springboot启动类上加上@ServletComponentScan，这样springboot启动时，就会先加载Servlet、Filter、Listener。 除了这种方法外，我们利用spring的配置类，还有一种解决办法，只不过这种方法不常用。

出现吧！html！——静态资源访问路径

我们知道，静态资源的访问也是一次请求，在我们之前用springMVC时，需要在xml配置文件中给做资源放行处理。在springboot中，静态资源访问是有默认访问的路径及顺序的，优先级由高到底为：

  1. classpath:META-INF/resources
  2. classpath:resources/
  3. classpath:static/
  4. classpath:public/

当然，我们还可以进行自定义资源访问路径

在application文件中配置中自定义资源路径： spring.resources.static-locations=路径1,路径2,路3…

properties文件：

spring.resources.static-locations= classpath:static/,classpath:test/

yml文件：

spring:
 resources:
   static-locations: calsspath:static/,classpath:test/

PS：自定义资源路径会将默认路径覆盖

小视频要放在云端——上传文件

springboot上传文件和springmvc一样

  1. HTML页面中，上传文件的格式必须为multipart/ form-data
    <form method="post" action="/fileUpdate1" enctype="multipart/form-data">
        <input type="file" name="file">
        <input type="submit" value="okok">
    </form>
  1. 接收的控制单元的参数中，直接接收MultipartFile类型参数即可
@RestController
public class FileUpdateController {
    @PostMapping("fileUpdate1")
    public String FileUpdate1(MultipartFile file) throws IOException {
        System.out.println(file.getOriginalFilename());
        file.transferTo(new File("A:/"+file.getOriginalFilename()));
        return "OK";
    }
}
