spring boot 的出现，改变了时代，改变了整个java后端的玩法

初学springboot，我将会将我的学习过程以及想法记录在这个系列中，方便以后查看，也方便初学者学习。如果有不对的地方，欢迎指正。

创建springboot项目——springboot，开箱即用

创建springboot项目是非常简单的，我们有三种方法来创建项目

  • 在spring官网创建后下载 创建好的项目会以压缩包的形式下载到本地。
  • 在IDEA中创建Spring Initializr项目
    这是最常用的方式，IDEA自动整合了spring官网创建的流程，省去了我们导入IDE的步骤，更加方便。
  • 在maven项目中继承springboot
    在pom.xml中需要添加三个代码块：继承、依赖、插件

springboot的启动入口——main

当我们用前两种方式创建项目后，在默认包下会有一个类，打开它我们惊奇的发现，它里面只有一个简单的main方法。

@SpringBootApplication
public class SpringbootDemo1Application {
    public static void main(String[] args) {
        SpringApplication.run(SpringbootDemo1Application.class, args);
    }
}

终于！我们的项目终于不依靠外部服务器来启动了，springboot内嵌了服务器，而且是多种，默认的是Tomcat。
这里我们还需要注意2点：

  • @SpringBootApplication
    这个注解的作用是什么？
  • SpringApplication.run()
    这个方法执行的内容是什么？

这两点疑惑我们以后肯定会明白，这里先疑问保留。

更加简单的配置文件——yaml

springboot出现的最大原因，就是诸多繁琐的配置文件，我们之前使用spring或者springmvc，都会配置很多xml文件，这些xml我们阅读起来非常不直观，而且每次创建项目都会有很多重复的地方，这其实是一种耦合。所以我觉得springboot的出现是必然的。 springboot将所有xml全部砍掉！我们再也不会配置xml了，大部分的配置全部由springboot自动为我们配置！
当然，我们还是需要提供给spring一些东西，比如数据库连接源。而这次，springboot使用了yaml/yml或者properties作为配置文件类型。

yml和xml YAML或者说YML是什么呢？它其实就是一个更人性化的配置文件，xml更加适合机器读写，而yml更适合人读写。 这篇博客给出了更详细的yaml介绍和基本用法https://www.cnblogs.com/sddai/p/9626392.html

yml和properties 这两种配置文件都书写简单，阅读直观，而相比之下，yaml/yml更加的直观，为什么这么说呢？因为YML有缩进。

在properties文件中，我们表示一个对象下的属性只能用“.”，比如：“server.port=8080”,这样我们看所有的配置全都是平级，配置项比较多，我们会阅读困难，而YML是下面这样： server:     port: 8080

配置文件的位置 因为springboot自动解析，所以配置文件的名字和位置都是不可随意更改的，文件名必须为：application.yml（properties同理）。那么我们可以将他放在哪个位置呢？

  • 项目根目录下 或 项目根目录/config下
  • classpath:下 或 classpath:config下

在我进行测试时发现，如果直接放到项目根或者根/config下，那么配置文件不会被编译输出到target文件夹下，但是它依然可以被解析，这是为什么呢？我查阅资料后依然没有明白，这里先进行疑问保留。
